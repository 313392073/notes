git中的push和pull命令都是针对远程操作的

pull从远程主机拉取资源
	命令：git pull <远程主机名> <远程分支名>:<本地分支名>
push推送资源到远端
	命令: git push <远程主机名> <本地分支名>:<远程分支名>
	
如下解释可方便记忆：
1、push或pull都是针对远程分支，所以开头都是远程分支名
2、':'前是当前想操作的分支
3、':'后边是目标分支名

## 分支合并

- 切换到主分支
- 使用：git merge feature_branch 将feature_branch分支合并进来。
- 冲突时需要手动解决冲突。大部分IDE会弹出合并窗口。
- 如果是两个分支差异较大，合并只想减少因在不同版本修复同一bug的代码，可能有很多冲突，手动修改还是合并有待考量。
